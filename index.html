🔄Syncthing F-Droid Verified

Peer-to-peer file synchronization. No cloud relay, no central server — devices discover each other on the LAN or via global discovery. Over NetBird, devices see each other on the same subnet (100.85.0.0/16), enabling direct P2P sync.

# On laptop:
sudo dnf install syncthing
systemctl --user enable --now syncthing

# Syncthing Web UI: http://127.0.0.1:8384
# Connect devices by scanning QR code or entering device ID.

Tip: On GrapheneOS, Syncthing runs as a foreground service. Grant "Ignore battery optimizations" permission for reliable background sync.

NETWORK Tier 2 — Network & Security

📡PCAPdroid F-Droid Verified

Network monitor and packet capture. Uses the Android VPN API (no root) to intercept all device traffic. Useful for debugging NetBird mesh connections, verifying that apps aren't leaking data, and inspecting DNS queries. Supports PCAP export for Wireshark analysis.

# Install from F-Droid.
# Start capture → select apps to monitor → view live connections.
# Export to PCAP for Wireshark: Share → Save to file.
Install sourceF-Droid (com.emanuelef.remote_capture) MethodVPN API — no root, no iptables NetBird✅ Works alongside NetBird (both use VPN interface — PCAPdroid can monitor NetBird traffic) ExportPCAP, NG, PCAPNG — open in Wireshark
Tip: On GrapheneOS, use PCAPdroid's "Always-on VPN" setting + block connections without VPN in Android settings for a firewall-like setup.
🌐NetBird (Android Client) Direct APK Verified

WireGuard-based mesh VPN. Connects the Pixel to the laptop over an encrypted P2P tunnel with no central server. The Android client joins the same NetBird network, gets a 100.85.x.x/16 IP, and can reach all peers directly.

# https://github.com/netbirdio/netbird/releases → netbird-client-*.apk
#
# After install:
# 1. Open NetBird app
# 2. Sign in (or use setup key)
# 3. Verify connection: Pixel gets 100.85.x.x address
# 4. Test: ping 100.85.29.98 from Termux
Install sourceGitHub Releases (APK direct download) ProtocolWireGuard via NetBird Cloud relay / NAT traversal Laptop IP100.85.29.98 (fedora.netbird.cloud) Pixel IP100.85.184.157 (dynamic, may change on reconnect) Latency~38ms P2P tunnel
Tip: On GrapheneOS, you can use NetBird's "Always-on VPN" in Android Settings → VPN → NetBird → Always-on VPN. This prevents any traffic from leaking outside the mesh.

Mesh Integration Details

NetBird mesh network: All devices get a 100.85.x.x/16 IP. Traffic is encrypted via WireGuard with NAT traversal. The laptop runs SSH (bound to mesh IP + localhost only), Syncthing, Ollama API, and the OpenCode AI agent.

Connecting from the Pixel

# From Termux on Pixel:

# 1. SSH into the laptop over mesh:
ssh q@100.85.29.98

# 2. Test connectivity:
ping -c 4 100.85.29.98
curl http://100.85.29.98:11434/api/tags

# 3. Forward a local port from laptop to Pixel (reverse tunnel):
# On laptop: ssh -R 8080:localhost:80 u0_aXXX@100.85.184.157

# 4. Set up Syncthing:
# - Laptop: http://127.0.0.1:8384
# - Pixel: Syncthing app → Add Device → enter laptop device ID

Security

# SSH is hardened on the laptop:
# - Listens ONLY on 100.85.29.98 and 127.0.0.1
